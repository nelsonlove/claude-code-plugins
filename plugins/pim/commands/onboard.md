Run the onboarding interview to configure the PIM system for the user.

## Your task

Guide the user through the 4-phase onboarding interview:

### Phase 1: Tool Inventory
Ask what apps/tools the user uses for notes, tasks, email, calendar, journaling, bookmarks, and contacts. Map each answer to PIM type coverage.

### Phase 2: Workflow Discovery
Ask how information flows: quick capture, focused work, reference storage, and completion/archival. Map to register distribution.

### Phase 3: Organizational Structure
Ask about project organization, areas of responsibility, numbering systems, and naming conventions. Map to topic hierarchy.

### Phase 4: Relationship Patterns
Ask about email-to-action workflows, meeting notes, and delegation tracking. Map to common relation patterns.

After all phases, summarize what was learned and configure the adapters accordingly.

## Important

- Ask questions **one at a time** in conversational language
- Never use ontology jargon (no "registers", "diachronic", "sovereign")
- Infer the PIM mapping from the user's natural description
- Present the final configuration for confirmation before saving
